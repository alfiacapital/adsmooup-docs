📘 AdsMooup API Documentation

Base URL: https://adsmooup-backend.vercel.app/api

All endpoints below (except /auth/login) require authentication. The server uses a JWT set as an HTTP‑only cookie via /auth/login.

🔐 Authentication

POST/auth/login

Authenticate user and set a JWT cookie.

FormData:
  • email – string
  • password – string

GET/auth/logout

Logout the user and clear the token cookie.

GET/auth/me

Get authenticated user.

👤 User Endpoints

GET/users

Returns all users.

POST/users

Create User.

FormData:
  • name – string
  • email – string
  • password – string

PUT/users/:id

Update a user by ID.

DELETE/users/:id

Delete a user by ID.

🍽️ Restaurant Endpoints

POST/restaurants

Step 1 — Create a new restaurant.

FormData (multipart):
  • name – string (required)
  • logo – file (image, required)

Uploads logo to Vercel Blob, returns restaurant with logoUrl and empty categories.

GET/restaurants

Get all restaurants.

GET/restaurants/:restaurantId

Get a specific restaurant by ID (with categories and menus).

PUT/restaurants/:restaurantId

Update restaurant info.

DELETE/restaurants/:restaurantId

Delete a restaurant.

POST/restaurants/:restaurantId/categories

Step 2 — Add a category to a restaurant.

FormData (multipart):
  • name – string (required)
  • image – file (image, required)

Stores { name, imageUrl, order, menus: [] } inside restaurant.categories.

POST/restaurants/:restaurantId/categories/:categoryId/menus

Step 3 — Add one or more menu images to a category (by index).

FormData (multipart):
  • images – multiple files (at least one required)

Each uploaded image becomes a menu item: { imageUrl, order }. Orders are appended sequentially.

PUT/restaurants/:restaurantId/categories/reorder

Update category order (drag & drop).

JSON body: 
{
  "order": [2, 0, 1] // array of existing indexes in the new desired order
}

Rebuilds categories based on the provided index mapping and normalizes each category’s order to its new position.

PUT/restaurants/:restaurantId/categories/:categoryId/menus/reorder

Update menu order within a category (drag & drop).

JSON body: 
{
  "order": [3, 1, 0, 2] // array of existing menu indexes in the new desired order
}

Rebuilds menus for the specified category and normalizes each menu’s order.

GET/restaurants/:restaurantId/completed

Step 4 — Completion helper (used by frontend to confirm and redirect).

Returns: { message: "Restaurant setup complete! Redirecting...", redirectTo: "/restaurants/:restaurantId" }

📢 Advertisement Endpoints

POST/ads

Create an ad linked to a restaurant and category.

FormData (multipart):
  • image – file (image)
  • restaurantId – string
  • categoryId – string

GET/ads

Get all ads.

GET/ads/:id

Get an ad by ID.

DELETE/ads/:id

Delete an ad by ID.
